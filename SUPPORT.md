# Support

## How to Get Help

Choose the path that best matches your request:

- Usage questions: open a GitHub issue and clearly mark it as a question if no dedicated discussion channel is available.
- Bug reports: use the bug report issue template and include a minimal reproduction.
- Feature ideas: use the feature request template and explain the problem you want to solve.
- Security concerns: do not open a public issue. Follow `SECURITY.md`.

## What to Include

Support requests are easier to answer when they include:

- Your Go version
- Your operating system
- The `Busen` version or commit you are using
- A minimal code sample
- The expected and actual behavior

## Project Scope

`Busen` is an in-process event bus for Go. Questions or requests related to persistence, replay, or distributed transport may be out of scope for the core library, even if they are valid use cases in adjacent systems.
